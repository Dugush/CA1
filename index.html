    Docker container

    Docker is set of platforms as a service which use OS to deliver software in a package named a container.
    Those containers are totally Isolated from each other, they are helping in bring software any where and they handle their software, configuration and libraries file.

    Docker Build

    In order to create docker file we need to use nano txt editor in cloud shell to open Dock file, we will write or past our text and close the nano text editor

    For the CA the instruction is only use for codes FROM, COPY, RUN and CMD.

  • From is used for docker hub to deliver the operating system and its version to the image
  • copy function is to copy the file from the file system or host and put the file in the image
  • To run a command on the image file we used the RUN command
  • CMD is used to function on the image , the command give information about the run time and the point of the entry. Docker Documentation

    • Example of the command


    FROM ubuntu: latest

    COPY script.sh /

    RUN chmod u+x /script.sh

    CMD /script.sh Docker Build
  • The next step is to save the Docker file by using CTRL+X then letter y and enter
  • Then we build the Dockerfile by using command docker build
  • Next, we will get an error due to the script.sh file is not created
  • We need to use nano text editor to create an script.sh file
  • following we add #! /bin/ and script message
  • We save the script file and build the docker image for the second time
  • We used command Docker images to show the images that created
  • This stage when we use command docker run image ID we will be able to print the message
  • We use command ps -a to see the container and image ID
